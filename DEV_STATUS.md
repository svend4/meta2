**Назначение**: автоматическая реконструкция (сборка) разорванных, разрезанных
газет, книг и документов из отсканированных фрагментов. Два ключевых алгоритма —
Танграм (геометрическая аппроксимация) и Фрактальная кромка (фрактальная
размерность + CSS-дескриптор) — создают уникальную «подпись» каждого края для
точного сопоставления фрагментов.

## 4. Алгоритмический конвейер (Pipeline)

```
              ┌─────────────┐
              │  Изображения │
              │  фрагментов  │
              └──────┬───────┘
                     │
           ┌─────────▼──────────┐
  Этап 1   │  Загрузка и        │  normalize_color()
           │  нормализация цвета│
           └─────────┬──────────┘
                     │
           ┌─────────▼──────────┐
  Этап 2   │  Сегментация       │  segment_fragment()
           │  (Otsu/Adaptive/   │  extract_contour()
           │   GrabCut)         │  estimate_orientation()
           └─────────┬──────────┘
                     │
           ┌─────────▼──────────┐
  Этап 3   │  Описание краёв    │  fit_tangram()
           │  Танграм + Фрактал │  compute_fractal_signature()
           │  → EdgeSignature   │  build_edge_signatures()
           └─────────┬──────────┘
                     │
           ┌─────────▼──────────┐
  Этап 4   │  Матрица           │  build_compat_matrix()
           │  совместимости     │  CSS + DTW + FD
           │  N×N               │
           └─────────┬──────────┘
                     │
           ┌─────────▼──────────┐
  Этап 5   │  Сборка            │  greedy_assembly()
           │  (Greedy / SA /    │  simulated_annealing()
           │   Beam / Gamma /   │  beam_search()
           │   Genetic / ACO /  │  gamma_optimizer()
           │   MCTS / Exhaust.) │
           └─────────┬──────────┘
                     │
           ┌─────────▼──────────┐
  Этап 6   │  OCR-верификация   │  verify_full_assembly()
           │  + отчёт           │
           └─────────┬──────────┘
                     │
              ┌──────▼───────┐
              │  Результат:  │
              │  Assembly +  │
              │  Score       │
              └──────────────┘
```

### 4.1 Методы сборки (assembly)

| Метод | Модуль | Описание |
|-------|--------|----------|
| **Greedy** | `greedy.py` | Жадная начальная сборка — быстрая, используется как seed |
| **Simulated Annealing** | `annealing.py` | Метаэвристическая оптимизация с температурным отжигом |
| **Beam Search** | `beam_search.py` | Поиск с ограниченной шириной луча (по умолчанию width=10) |
| **Gamma Optimizer** | `gamma_optimizer.py` | Оптимизация с гамма-коррекцией скоров |
| **Genetic Algorithm** | `genetic.py` | Генетический алгоритм (кроссовер + мутация) |
| **Ant Colony** | `ant_colony.py` | Муравьиная оптимизация (ACO) |
| **MCTS** | `mcts.py` | Monte Carlo Tree Search |
| **Exhaustive** | `exhaustive.py` | Полный перебор (для малых задач) |

### 4.2 Методы сопоставления краёв (matching)

| Метод | Модуль | Описание |
|-------|--------|----------|
| **DTW** | `dtw.py` | Dynamic Time Warping с окном Сакое-Чибы |
| **CSS** | (через `synthesis.py`) | Curvature Scale Space (MPEG-7) |
| **SIFT** | `feature_match.py` | Сопоставление по ключевым точкам |
| **ICP** | `icp.py` | Iterative Closest Point |
| **Color** | `color_match.py` | Сопоставление по цвету |
| **Texture** | `texture_match.py` | Текстурное сопоставление |
| **Geometric** | `geometric_match.py` | Геометрическое сопоставление |
| **Spectral** | `spectral_matcher.py` | Спектральное сопоставление |
| **Graph** | `graph_match.py` | Графовое сопоставление |
| **Consensus** | `consensus.py` | Консенсус нескольких методов |

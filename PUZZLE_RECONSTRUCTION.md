## 1. Постановка задачи

### Входные данные
- N фрагментов разорванного документа (газеты, книжного листа, письма)
- Каждый фрагмент уникален по форме: края образуют неповторимую «береговую линию»
- Внутри фрагмента находится часть исходного текста / изображения

### Цель
Автоматически восстановить исходный документ: найти правильную ориентацию и
взаимное расположение всех фрагментов так, чтобы их края совпали, а текст
образовал непрерывное целое.

### Ключевое наблюдение
Когда бумагу рвут, каждая граница разрыва **уникальна** — она никогда не
повторяется. Это свойство позволяет сопоставлять фрагменты по форме края,
даже если текст не читаем или отсутствует.

### Сложность задачи
- При N фрагментах число возможных конфигураций: **O(N! × 4^N)** — NP-полная задача
- Требуется эффективное представление краёв и быстрое попарное сравнение

## 3. Алгоритм 1 — Танграм-аппроксимация внутренней геометрии

### Идея
Танграм — китайская головоломка из 7 фигур (2 большой треугольник, 1 средний,
2 малых, 1 квадрат, 1 параллелограмм). Любую плоскую фигуру можно приближённо
разложить в танграм. Мы используем это свойство для нормализации формы фрагмента.

### Шаги

```
1. Сегментация фрагмента
   └─ Бинаризация (Otsu threshold)
   └─ Морфологическое закрытие (remove_small_holes)
   └─ Нахождение самого большого контура → маска фрагмента

2. Минимальная вписанная геометрическая фигура
   └─ Convex Hull фрагмента
   └─ Аппроксимация Рамера-Дугласа-Пекера (RDP) → упрощённый полигон
   └─ Классификация формы: треугольник / прямоугольник / параллелограмм /
      трапеция / многоугольник (5-8 вершин)

3. Разложение в танграм-компоненты
   └─ Алгоритм покрытия: максимальный танграм-контур ВНУТРИ фрагмента
   └─ Остаток (фрагмент минус танграм) → «береговая зона»
   └─ Выход: T_contour — упорядоченный список вершин танграм-полигона

4. Нормализация
   └─ Центроид танграма → начало координат
   └─ Главная ось инерции → ось X
   └─ Масштаб: диагональ описанного прямоугольника = 1.0
```

### Результат модуля 2А
```python
T_contour = [(x0, y0), (x1, y1), ..., (xk, yk)]  # Нормализованный полигон
T_axes    = (angle, scale)                         # Ориентация и масштаб
T_class   = "rectangle" | "triangle" | ...        # Тип фигуры
```

## 4. Алгоритм 2 — Фрактальная береговая линия края

### Идея
Разорванный край бумаги ведёт себя как «фрактальная береговая линия»:
при увеличении масштаба появляются всё новые детали. Фрактальная размерность (FD)
и форма кривой при разных масштабах задают уникальную «подпись» каждого края.

### Два метода вычисления FD

#### Метод 1: Box-counting
```
1. Наложить сетку с размером ячейки r на контур
2. Подсчитать N(r) — число занятых ячеек
3. Уменьшать r: r = r_max / 2^k, k = 1..K
4. FD = -slope( log N(r) vs log r )
```

#### Метод 2: Divider (Ричардсон / компас)
```
1. Задать длину «линейки» s
2. Пройти вдоль контура шагами длиной s, подсчитать число шагов L(s)
3. Повторить для s = s_max / 2^k
4. FD = -slope( log L(s) vs log s )
   (Парадокс Ричардсона: чем мельче линейка, тем длиннее «берег»)
```

### Фрактальная интерполяционная функция (FIF)
Метод Барнсли — точное воспроизведение формы кромки через IFS-аттрактор:

```
1. Взять N опорных точек на краю: {(t_i, y_i)}
2. Определить систему аффинных отображений (IFS):
   w_n(x, y) = (a_n·x + e_n,  c_n·x + d_n·y + f_n)
3. Итеративно подобрать вертикальные сжатия d_n
   до совпадения аттрактора IFS с реальным краем
4. Коэффициенты {d_n} — компактный уникальный «отпечаток» края
```

### Результат модуля 2Б
```python
F_signature = {
    "fractal_dim":    1.37,               # FD по Divider
    "fd_box":         1.41,               # FD по Box-counting
    "ifs_coeffs":     [d_1, d_2, ..., d_M],  # FIF-коэффициенты
    "css_descriptor": [...],              # Curvature Scale Space вектор
    "chain_code":     "0112233...",       # Цепной код Фримана
}
```

## 5. Синтез: Уникальная подпись кромки

### Ключевая идея (двойная перспектива)

```
            ВНУТРИ фрагмента          СНАРУЖИ фрагмента
                    │                        │
           Танграм-контур              Фрактальная кривая
           (геометрически              (форма «волнореза»,
            правильный,                 или «камень в море»)
            упрощённый)
                    │                        │
                    └───────────┬────────────┘
                                │
                         ЗОНА НАХЛЁСТА
                         (береговая зона)
                                │
                    ┌───────────┴────────────┐
                    │  «Золотая середина»    │
                    │  — средневзвешенная    │
                    │    линия пересечения   │
                    │  = EdgeSignature[e]    │
                    └────────────────────────┘
```

### Математическое описание

Пусть `B_T(t)` — граница танграм-полигона, параметризованная по дуге `t ∈ [0,1]`.
Пусть `B_F(t)` — фрактальная кривая края (FIF-аттрактор).

Виртуальная кромка:
```
B_virtual(t) = α · B_T(t) + (1 - α) · B_F(t),   α ∈ [0.3, 0.7]
```

Уникальная подпись:
```
EdgeSignature[e] = {
    B_virtual(t),        # Параметрическая кривая
    FD(e),               # Фрактальная размерность
    CSS(e),              # Curvature Scale Space
    IFS_coeffs(e),       # FIF-коэффициенты
    tangram_class,       # Класс геометрии
}
```

### Почему это даёт уникальность?
- Танграм-аппроксимация стабилизирует крупную геометрию (не зависит от мелких неровностей)
- FIF точно воспроизводит мелкие детали разрыва на любом масштабе
- Их наложение порождает описание, инвариантное к освещению и сканированию,
  но **максимально чувствительное** к реальной форме края

## 6. Алгоритм сопоставления фрагментов

### Попарная совместимость

Для каждой пары (край `e_i` фрагмента `i`, край `e_j` фрагмента `j`):

```
MatchScore(e_i, e_j) = w1 · CSS_sim(e_i, e_j)
                     + w2 · DTW_dist(B_i, B_j)
                     + w3 · FD_diff(e_i, e_j)
                     + w4 · Text_coherence(i, j)
```

Где:
- **CSS_sim** — косинусное сходство Curvature Scale Space векторов
- **DTW_dist** — Dynamic Time Warping между параметрическими кривыми краёв
  (важно: сопрягаемые края при совпадении должны быть «зеркальными»)
- **FD_diff** — разница фрактальных размерностей (малая = хорошо)
- **Text_coherence** — оценка OCR: насколько текст «читается» через стык

### Условие совместимости краёв

Если фрагменты `i` и `j` были ранее соседями в оригинале, то их соприкасающиеся
края `e_i` и `e_j` являются взаимно-дополняющими:

```
B_i(t) + B_j(1-t) ≈ constant   (кривые зеркальны)
FD(e_i) ≈ FD(e_j)               (размерности совпадают)
CSS(e_i) ≈ mirror(CSS(e_j))     (подписи симметричны)
```

### Матрица совместимости

```python
CompatMatrix[N_edges × N_edges]:
    for each edge e_i of fragment i:
        for each edge e_j of fragment j (j ≠ i):
            score = MatchScore(e_i, e_j)
            if score > threshold:
                CompatMatrix[e_i][e_j] = score
```

## 7. Глобальная сборка

### Постановка как задача оптимизации

```
Найти: расстановку P = {(x_i, y_i, θ_i)} для всех фрагментов i
Максимизировать: Σ_{смежные (i,j)} MatchScore(e_i, e_j)
Ограничения:
    - Фрагменты не перекрываются
    - Итоговый документ вписывается в ожидаемый формат
```

### Методы оптимизации

#### Жадный алгоритм (быстрый старт)
```
1. Взять наиболее уверенное совпадение краёв → разместить первую пару
2. Расширять сборку: искать лучший следующий фрагмент к уже размещённым
3. Повторять до исчерпания фрагментов
Сложность: O(N^2 × K) где K — среднее число краёв фрагмента
```

#### Имитация отжига (Simulated Annealing)
```
T = T_max
while T > T_min:
    выбрать случайный фрагмент → переместить / повернуть
    ΔE = новый_score - старый_score
    принять если ΔE > 0 или random() < exp(ΔE/T)
    T = T × cooling_rate
```

#### Стохастическая оптимизация с гамма-распределением (2026)
Моделирует отклонения краёв гамма-распределением:
```
deviation ~ Gamma(k, θ)
likelihood = Π Gamma_pdf(|edge_i(t) - edge_j(1-t)|; k, θ)
Максимизировать суммарное правдоподобие по всем стыкам
```

## 8. Псевдокод

```python
# ============================================================
# ГЛАВНАЯ ПРОГРАММА: Восстановление разорванного документа
# ============================================================

function RECONSTRUCT_DOCUMENT(scan_images: List[Image]) -> Document:

    # ЭТАП 1: Предобработка
    fragments = []
    for img in scan_images:
        mask     = segment_fragment(img)        # Бинаризация + контур
        contour  = extract_contour(mask)        # Упорядоченный список точек
        orient   = estimate_orientation(img)    # По строкам текста
        fragment = Fragment(img, mask, contour, orient)
        fragments.append(fragment)

    # ЭТАП 2: Описание каждого фрагмента
    for frag in fragments:
        # Алгоритм 1: Танграм
        frag.tangram = fit_tangram(frag.contour)
        #   └─ convex_hull → rdp_simplify → classify_shape
        #   └─ inscribe_tangram_inside → T_contour

        # Алгоритм 2: Фрактальная кромка
        frag.fractal = compute_fractal_signature(frag.contour)
        #   └─ box_counting_fd(contour) → FD_box
        #   └─ divider_fd(contour)      → FD_divider
        #   └─ fit_ifs(contour)         → ifs_coeffs
        #   └─ css_descriptor(contour)  → css_vec

        # Синтез: Виртуальная кромка
        for edge in frag.edges:  # обычно 4-8 краёв на фрагмент
            T_edge = tangram_edge_segment(frag.tangram, edge)
            F_edge = fractal_edge_segment(frag.fractal, edge)
            edge.signature = blend_signatures(T_edge, F_edge, alpha=0.5)

    # ЭТАП 3: Матрица совместимости
    N_edges = sum(len(f.edges) for f in fragments)
    compat  = zeros(N_edges, N_edges)

    for (e_i, e_j) in all_edge_pairs(fragments):
        if e_i.fragment == e_j.fragment:
            continue
        score = match_score(e_i, e_j)
        compat[e_i.id][e_j.id] = score

    # ЭТАП 4: Глобальная сборка
    placement = greedy_assembly(fragments, compat)   # Начальное решение
    placement = simulated_annealing(placement, compat, T_max=1000)

    # ЭТАП 5: Верификация
    ocr_score = verify_text_continuity(placement)
    if ocr_score < threshold:
        placement = beam_search(fragments, compat, width=10)

    return render_document(placement)


# ============================================================
# ВСПОМОГАТЕЛЬНЫЕ ФУНКЦИИ
# ============================================================

function fit_tangram(contour):
    hull     = convex_hull(contour)
    polygon  = rdp_simplify(hull, epsilon=0.02)
    shape    = classify_shape(polygon)  # "triangle", "rectangle", etc.
    tangram  = inscribe_tangram(polygon, shape)
    return normalize(tangram)  # центроид в 0, ось X по главной оси

function compute_fractal_signature(contour):
    fd_box  = box_counting(contour, r_min=1, r_max=64)
    fd_div  = divider_method(contour, s_min=1, s_max=64)
    ifs     = fit_ifs_coefficients(contour, n_transforms=8)
    css     = curvature_scale_space(contour, sigma_range=[1..16])
    chain   = freeman_chain_code(contour)
    return FractalSignature(fd_box, fd_div, ifs, css, chain)

function match_score(e_i, e_j):
    # Края должны быть зеркальными при совпадении
    e_j_rev = reverse(e_j.signature.curve)

    css_sim  = cosine_similarity(e_i.signature.css, e_j.signature.css)
    dtw_dist = dynamic_time_warping(e_i.signature.curve, e_j_rev)
    fd_diff  = abs(e_i.signature.fd - e_j.signature.fd)
    text_coh = ocr_coherence(e_i.fragment, e_j.fragment)

    return 0.35 * css_sim
         + 0.30 * (1 / (1 + dtw_dist))
         + 0.20 * (1 / (1 + fd_diff))
         + 0.15 * text_coh

function blend_signatures(T_edge, F_edge, alpha):
    # «Золотая середина» танграма и фрактала
    virtual_curve = alpha * T_edge.curve + (1 - alpha) * F_edge.curve
    return EdgeSignature(
        curve    = virtual_curve,
        fd       = (T_edge.fd + F_edge.fd) / 2,
        css      = weighted_mean(T_edge.css, F_edge.css, alpha),
        ifs      = F_edge.ifs_coeffs,  # фрактал точнее для мелких деталей
    )
```

### Пример: Вычисление фрактальной размерности (Box-counting)

```python
import numpy as np
from numba import jit

@jit(nopython=True)
def box_counting_fd(contour_points: np.ndarray,
                    r_min: int = 1,
                    r_max: int = 128) -> float:
    """
    Вычисляет фрактальную размерность контура методом Box-counting.

    Args:
        contour_points: массив (N, 2) координат точек контура
        r_min, r_max:   диапазон размеров ячеек сетки

    Returns:
        FD: фрактальная размерность (1.0 = гладкая линия, 2.0 = заполняет плоскость)
    """
    log_r = []
    log_N = []

    r = r_max
    while r >= r_min:
        # Нормируем координаты в сетку размера r
        grid_x = (contour_points[:, 0] / r).astype(np.int32)
        grid_y = (contour_points[:, 1] / r).astype(np.int32)

        # Уникальные ячейки
        cells = set(zip(grid_x, grid_y))
        N = len(cells)

        log_r.append(np.log(1.0 / r))
        log_N.append(np.log(N))

        r //= 2

    # Линейная регрессия: FD = slope(log N vs log(1/r))
    log_r = np.array(log_r)
    log_N = np.array(log_N)
    coeffs = np.polyfit(log_r, log_N, 1)
    return coeffs[0]  # наклон = FD
```

### Пример: CSS-дескриптор (MPEG-7)

```python
import numpy as np
from scipy.ndimage import gaussian_filter1d

def curvature_scale_space(contour: np.ndarray,
                          sigmas: list = None) -> np.ndarray:
    """
    Curvature Scale Space — стандарт MPEG-7 для описания формы.
    Возвращает матрицу нулей кривизны при разных масштабах.
    """
    if sigmas is None:
        sigmas = [2**k for k in range(7)]  # 1, 2, 4, 8, 16, 32, 64

    x = contour[:, 0].astype(float)
    y = contour[:, 1].astype(float)
    css_image = []

    for sigma in sigmas:
        # Сглаживание контура гауссовым фильтром
        Xs = gaussian_filter1d(x, sigma, mode='wrap')
        Ys = gaussian_filter1d(y, sigma, mode='wrap')

        # Производные
        Xs1 = np.gradient(Xs)
        Xs2 = np.gradient(Xs1)
        Ys1 = np.gradient(Ys)
        Ys2 = np.gradient(Ys1)

        # Кривизна
        denom = (Xs1**2 + Ys1**2)**1.5
        kappa = (Xs1 * Ys2 - Xs2 * Ys1) / (denom + 1e-10)

        # Нули кривизны (точки перегиба)
        zero_crossings = np.where(np.diff(np.sign(kappa)))[0]
        css_image.append((sigma, zero_crossings))

    return css_image
```
